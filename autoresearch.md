# Autoresearch: opencli ChatGPT Control

## Objective
Maximize opencli's ChatGPT control surface. Each new working command file in
`src/clis/chatgpt/` that compiles without TypeScript errors counts as a feature.
Current baseline: 7 command files (ask, deepresearch, new, promode, read, send, status).
ax.ts is excluded (utility module, not a command).

## Metrics
- **Primary**: feature_count (count, higher is better)
- **Secondary**: typecheck_errors (count, lower is better)

## How to Run
`./autoresearch.sh` — outputs `METRIC name=number` lines.

## Files in Scope
- `src/clis/chatgpt/*.ts` — all ChatGPT command implementations
- `src/clis/chatgpt/ax.ts` — shared utilities (getVisibleChatMessagesFromPage, etc.)
- `dist/cli-manifest.json` — DO NOT edit directly; regenerated by `npm run build`

## Off Limits
- `src/engine.ts`, `src/registry.ts`, `src/types.ts` — core framework, don't touch
- Other site clis (bilibili, discord, etc.)
- Test files

## Constraints
- All new commands must compile with `npx tsc --noEmit` (zero errors)
- Each command must export a `*Command` constant using the `cli()` factory
- Follow pattern: `import { cli, Strategy } from '../../registry.js'`
- Use `IPage` from `'../../types.js'` for the page parameter
- For CDP-level operations, import CDPBridge from `'../browser/cdp.js'` (but prefer IPage methods)
- Commands using the browser: `browser: true`, `strategy: Strategy.PUBLIC`
- Linux only (no macOS AppleScript paths needed)

## Architecture Notes
- `CDPBridge.connect()` creates an isolated tab via `PUT /json/new` — good for commands
  that need fresh context
- `getVisibleChatMessagesFromPage(page)` — reads `[data-message-author-role="assistant"]`
- `selectChatGPTMode(bridge, mode)` — switches model via CDP mouse events
- `isGenerating(page)` — checks for stop button presence
- build-manifest auto-scans all .ts files in src/clis/chatgpt/ — new commands auto-register
- The manifest's `timeout` field controls framework kill timeout; `timeoutSeconds` in cli()
  is also extracted by build-manifest.ts

## What's Been Tried
### Baseline (run 1): 7 features
Starting point: ask, deepresearch, new, promode, read, send, status.

### Run 2: conversations — list sidebar conversations
Added conversations.ts: extracts `nav a[href*="/c/"]` from sidebar.
Optional index arg to navigate to a specific conversation.

### Run 3: history — read full conversation
Added history.ts: reads ALL messages with `[data-message-author-role]` selector.
Returns role + text for each message turn.

### Run 4: wait — wait for generation
Added wait.ts: polls stop-button presence until it disappears.

### Run 5: stop — stop generation
Added stop.ts: clicks `[data-testid="stop-button"]` or any "Stop" button.

### Run 6: switch — expose model switching as CLI
Added switch.ts: thin wrapper around selectChatGPTMode() from promode.
Models: instant, thinking, pro (or gpt-4o, o3, o1 aliases).

### Run 7: image — DALL-E image generation
Added image.ts: navigates to chatgpt.com, sends prompt, polls for image URL
from `[data-testid*="image"]` or `img[alt*="DALL"]`.

### Run 8: projects — list/open ChatGPT projects
Added projects.ts: extracts projects from sidebar nav links matching /g/ pattern.

### Run 9: share — get conversation share URL
Added share.ts: clicks share button, extracts URL from modal.

### Run 10: goto — navigate to conversation URL
Added goto.ts: simple page.goto() wrapper with validation.

### Run 11: upload — upload file to conversation
Added upload.ts: uses CDP Runtime.evaluate to trigger file input, then CDP
Page.handleFileChooser + Input.setFileInputFiles to inject file path.
